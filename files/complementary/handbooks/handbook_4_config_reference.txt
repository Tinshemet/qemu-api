  QEMU-API  —  CONFIGURATION REFERENCE  (handbook part 4)

Every configuration file, every key, every environment variable, and every
default value. Where env vars override config keys, that is documented too.

────────────────────────────────────────────────────────────────────────────────
1. ENVIRONMENT VARIABLES
────────────────────────────────────────────────────────────────────────────────

All env vars are read at import time (module level). They override the
corresponding config.json values.

SERVER_URL
  Default:  "http://localhost:8080"  (from client/connection_config.json)
  Values:   "http://host:port" | "https://host:port"
  Effect:   URL of the qemu-api server for the chat client to connect to.
  Read by:  client/ui/chat_client.py

API_TOKEN
  Default:  "" (from connection_config.json on each side)
  Values:   Any string (recommended: 32+ random chars)
  Effect:   Shared secret for Bearer authentication.
            Client chat_client: sent in Authorization header.
            Server api_server: compared against incoming tokens.
  Read by:  client/ui/chat_client.py, server/http/api_server.py
  Also read from (server side): ~/.qemu-api.token (file fallback)

API_TIMEOUT
  Default:  120 (from connection_config.json)
  Values:   Integer (seconds)
  Effect:   HTTP request timeout for /chat and /execute calls.
  Read by:  client/ui/chat_client.py, server/executor_client.py

API_CA_CERT
  Default:  None
  Values:   Absolute path to a PEM-format CA certificate file
  Effect:   When set, the client uses this CA to verify the server's TLS cert.
            Used for self-signed certificates.
            When unset, system CA bundle is used (correct for Let's Encrypt).
  Read by:  client/ui/chat_client.py

API_VERIFY_SSL
  Default:  "1" (verify enabled)
  Values:   "1" (verify) | "0" (skip verification)
  Effect:   "0" disables TLS certificate verification entirely.
            NEVER use in production — only for local development.
  Read by:  client/ui/chat_client.py

OLLAMA_MODEL
  Default:  "qwen2.5:7b" (from server/ai/config.json → ollama_model)
  Values:   Any Ollama model name ("llama3.1:8b", "mistral", etc.)
  Effect:   Which model is used for all AI calls.
  Read by:  server/ai/ollama_client.py

OLLAMA_URL
  Default:  "http://localhost:11434" (from server/ai/config.json → ollama_url)
  Values:   Any URL where Ollama is reachable
  Effect:   Where the Ollama API server is at (always server-local).
  Read by:  server/ai/ollama_client.py

────────────────────────────────────────────────────────────────────────────────
2. server/connection_config.json  (read by server/executor_client.py and api_server.py)
────────────────────────────────────────────────────────────────────────────────

{
  "url":   "local",
    # Always "local" — server runs QEMU co-located.

  "token": "",
    # API token. Match this to API_TOKEN env var or ~/.qemu-api.token.

  "client_allowed_vms": [],
    # VMs visible and accessible to clients. Empty = all VMs allowed.
    # Non-empty = allowlist. Hidden VMs return "not found" to clients (opsec).

  "client_allowed_profiles": []
    # Hardware profiles visible to clients. Empty = all profiles allowed.
}

ACCESS CONTROL ENFORCEMENT:
  Two-layer enforcement — clients cannot bypass either layer:
  1. GET /sync filters the VM/profile inventory before the client sees it.
  2. executor_client.execute_tool() blocks tool calls targeting hidden VMs
     for both direct /execute calls AND AI-initiated calls through /chat.

3. client/connection_config.json  (read by client/ui/chat_client.py)
────────────────────────────────────────────────────────────────────────────────

{
  "server_url": "http://localhost:8080",
    # URL of the qemu-api server. Override with SERVER_URL env var.

  "token": "",
    # API token. Leave empty and use API_TOKEN env var.

  "ca_cert": null,
    # Path to a CA cert PEM file for self-signed HTTPS servers.
    # null → use system CA bundle (correct for Let's Encrypt).

  "verify_ssl": true,
    # Set false only for development with self-signed certs without ca_cert.

  "timeout": 120
    # HTTP request timeout in seconds.
}

3. shared/executioner/config.json  (read by shared/executioner/tool_executor.py)
────────────────────────────────────────────────────────────────────────────────

{
  "executor": {
    "url": "local",
      # Always "local" on the server — QEMU and AI are co-located.

    "port": 8080,
      # Port the API server listens on. Used by setup scripts.

    "timeout": 120,
      # Timeout for tool execution operations.

    "allowed_remote_tools": [
      "create_vm", "launch_vm", "stop_vm", "list_vms", "vm_status",
      "delete_vm", "clone_vm", "resize_disk",
      "snapshot_create", "snapshot_list", "snapshot_restore", "snapshot_delete",
      "check_system", "list_profiles", "check_profile_compatibility",
      "fingerprint_vm", "check_disk", "revert",
      "generate_guest_setup", "mark_stealth_done",
      "create_cluster", "delete_cluster", "list_clusters"
      # Tools NOT in this list are blocked at the HTTP layer (403 Forbidden).
      # send_monitor_cmd is intentionally excluded — raw QEMU monitor access
      # from an unauthenticated remote connection is a security risk.
    ]
  },

  "create_vm_defaults": {
    "disk_size_gb": 40,      # default disk size for new VMs
    "disk_format": "qcow2",  # default disk format
    "disk_bus": "virtio",    # default disk controller bus
    "network_mode": "nat",   # default network mode
    "bridge": "virbr0"       # default bridge interface
  },

  "tool_defaults": {
    "snap_name": "snap1",    # default snapshot name when not specified
    "log_lines": 50          # default number of log lines for get_vm_logs
  },

  "valid_machine_types": ["q35", "pc", "virt", "microvm", "raspi3b", "raspi4b"],
    # Machine types that are accepted by the sanitizer and executor.
    # Profile names are NOT in this list — they are stripped before reaching here.

  "arm_cpu_prefixes": ["cortex-a", "cortex-r", "cortex-m", "neoverse", "a64fx"]
    # CPU model name prefixes that indicate an ARM CPU. Used by the sanitizer
    # to detect ARM CPUs being used on x86_64 VMs (and replace with "host").
}

3. server/ai/config.json
────────────────────────────────────────────────────────────────────────────────

Owned by the provider machine (AI side). Read by cli.py and ollama_client.py.

{
  "ollama_url": "http://localhost:11434",
    # Ollama server URL. Override with OLLAMA_URL env var.

  "ollama_model": "qwen2.5:7b",
    # Default model. Override with OLLAMA_MODEL env var.

  "tool_loop_max": 6,
    # Maximum number of tool calls the AI can chain in one user prompt.
    # Prevents infinite loops. After this limit, the loop breaks.

  "tool_loop_max_override": null,
    # Override for tool_loop_max. If set (not null), this value is used instead.

  "auto_clear_session": true,
    # If true, the session history is auto-cleared when drift is detected
    # (AI's remembered VMs don't match actual VM list).

  "exit_commands": ["exit", "quit", "bye", "q", ":q", "done"],
    # Any of these in the chat loop exits the program.

  "shortcut_commands": {
    "list": "list_vms",
    "profiles": "list_profiles",
    "system": "check_system",
    "clear session": "clear_session",
    "exit": "exit"
    # Bypass Ollama entirely for these common commands.
  },

  "action_words": ["create", "make", "launch", "start", "run", "stop", ...],
    # Words that suggest the user wants to perform an action.
    # Used to decide whether to call Ollama or just describe.

  "os_keywords_gate": ["linux", "windows", "ubuntu", "mint", "windows", ...],
    # OS-related keywords used by the context gate.

  "confirm_yn": { ... },
    # Patterns for detecting yes/no answers from the user.

  "confirm_name": { ... }
    # Patterns for detecting VM name inputs from the user.
}

6. shared/sanitizer/context_gate_config.json
────────────────────────────────────────────────────────────────────────────────

Defines required fields per tool for the context gate.

{
  "create_vm": {
    "required": ["name"],
    "question": "What would you like to name this VM?",
    "options": ["my-windows-vm", "dev-machine", "test-ubuntu"]
  },
  "launch_vm": {
    "required": ["name"],
    "question": "Which VM would you like to launch?",
    "options": []    # filled dynamically from current VM list
  },
  ...
}

If a required field is missing or empty after sanitization, gate_check()
returns the question and options as a clarify response.

8. shared/api/config.json

{
  "vm_base_dir": "~/.qemu_vms",
    # Root directory for all VM data. Expand ~ at runtime.

  "qemu_binary": "qemu-system-x86_64",
    # Default QEMU binary. Overridden to qemu-system-aarch64 for ARM VMs.

    # qemu-img binary for disk operations.

  "qmp_timeout": 7.5,
    # Seconds to wait for QMP socket to appear after QEMU starts.
    # 15 retries × 0.5s = 7.5s max.

  "qmp_retries": 15,
    # Number of QMP connection attempts.

  "qmp_retry_delay": 0.5,
    # Delay between QMP retries in seconds.

  "vnc_password_expire_seconds": 600,
    # VNC one-time password validity in seconds (10 minutes).

  "launch_log_lines": 100,
    # Number of launch.log lines to keep (older lines are truncated).

  "session_max_messages": 40
    # Maximum messages to keep in .session.json.
}

9. MachineConfig FIELDS AND DEFAULTS
────────────────────────────────────────────────────────────────────────────────

This is every field in the MachineConfig dataclass, its type, default value,
and what it controls in the QEMU command.

  name             str   (required)       VM directory name, used everywhere
  os_type          str   "linux"          Affects disk bus (sata for windows), UEFI auto
  os_name          str   ""               Distro name for ISO auto-detection
  description      str   ""               Human-readable note, not used in QEMU args
  memory_mb        int   4096             -m {memory_mb}
  cpu_cores        int   2                -smp {cpu_cores},threads={cpu_threads}
  cpu_threads      int   1                -smp cores,threads=
  cpu_model        str   "host"           -cpu {cpu_model}[,flags]
  machine_type     str   "q35"            -machine type={machine_type}
  machine_arch     str   "x86_64"         selects qemu binary
  qemu_binary      str   "qemu-system-x86_64"  the QEMU executable
  display          str   "sdl"            -display {display} or -vga
  gpu              str   "virtio-vga"     -device {gpu}
  audio            str   "hda"            -audiodev {audio} -device intel-hda
  bios             str   "seabios"        "ovmf" or "ovmf_ms" → pflash drive
  uefi             bool  False            enables OVMF firmware path
  kvm              bool  True             -enable-kvm when True
  tpm              bool  False            starts swtpm, adds -tpmdev/-device
  stealth          bool  False            applies _apply_stealth(), implies hardened
  hardened         bool  False            applies _harden() (hide KVM CPUID)
  battery          bool  False            -device acpi-battery
  hugepages        bool  False            -mem-path /dev/hugepages
  balloon          bool  True             -device virtio-balloon when True
  manufacturer     str   ""              -smbios type=1,manufacturer=
  product_name     str   ""              -smbios type=1,product=
  serial_number    str   ""              -smbios type=1,serial=
  board_product    str   ""              -smbios type=2,product=
  bios_vendor      str   ""              -smbios type=0,vendor=
  bios_version     str   ""              -smbios type=0,version=
  smbios_type      str   ""              human label → chassis_type_byte lookup
  machine_class    str   ""              "laptop" → chassis_type_byte=9 etc.
  networks         list  [NetworkConfig] one -netdev/-device pair per entry
  disks            list  [DiskConfig]    one -drive per entry
  iso_path         str   ""              -drive if=cdrom,file=
  extra_args       list  []              appended verbatim to QEMU command
  rtc_clock        str   "utc"           -rtc base={rtc_clock}
  rtc_driftfix     str   "slew"          -rtc driftfix={rtc_driftfix}
